Программа предназначина для управления веб окружением Bitix24 запущеным в виртуальной среде, на VDS или физическом сервере.
Сейчас программа работает через powershell, но у powershel по умолчанию запрещены использование скриптов в windows без цифровой подписи,
придется включать использование скриптов вручную. Потом попробую настроить авто включение скриптов или подписать свой скирпт правильной подписью.

Начало работы:
Для работы с данным скриптом в windows 7 (возможно для 8ки тоже потребуется) необходимо скачать и установить Windows Management Framework 4.0
Скачать можно тут https://www.microsoft.com/en-us/download/details.aspx?id=40855

В коде файла в строках 12 и 13 укажите переменные своих ключей. Ключи должны быть в формате OPENSSH. Путь к ключам моет быть казан как относительно каталога запуска скрипта ".\путь\файл.key" , так и полный путь "С:\путь\файл.key" (Ковычки обязательны.
Строка 12 ключ без парольной фразы, используется для настройки в дайльнейшем
Строка 13 ключ с парольной фразой, будет спользован в пунктеменю №9 после окончательной настройки (ПОКА НЕ РАБОТАЕТ)
$sshkeynopass = ".\путь\файл.key";
$sshkeyyespass = ".\путь\файл.key";

1. Указать Пользователя и HOST
  Необходимо указывать в формате user@host. Юзера указываем обычно root и ип адрес или DNS имя сервера на котором установлен Битрикс       коробка. Подходит как для виртуальных машин прелоставляемых Битрикс24 так и для CENTOS7. На данный момент не проверялась работа с       UBUNTU и DEBiAN ( скорее всего работать в данных системах не будет).
2. Установить SSH key на сервер(надо знать пароль ROOT)
  Появляется приглашение для выбора файла публичного SSH ключа (.pub). После выбора файла появится предложение ввода пароля root для         установленной системы Битрикс24 коробка. SSHkey будет скопирован в /root/.ssh/authorized_keys
3. Установить дополнительный софт
  Устанавливается (mc, nano, net-tools, cifs-utils, htop)
4. Проверить сервисы Веб-окружения
  Выводит список демонов со статусом состояния на данный момент проверяет nginx, mysql, apache
5. Проверить Backup
  Выводит список файлов в папке /home/bitrix/www/bitrix/backup . Это стандартный путь где битрикс хранит бекапы.
6. Использование ОЗУ
  Выводит информацию о использованной/общей оперативной памяти.
7. Свободное место на дисках
  Выводит информацию о использованной/общей памяти всех дисков.
8. Перезагрузить удаленный host
  тут понятно перезагрузка
9. Заменить SSH ключ support1 на itexto (Не работает)
  В разработке
0. Выход
  Закрывает окно скрипта
